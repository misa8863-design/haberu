# はべる / HABERU LP 制作仕様

## GitHubアップロード用メモ

このフォルダは、そのままGitHubへアップロードできる静的LP一式です。

アップロードする主なファイル:

- `index.html`
- `style.css`
- `script.js`
- `assets/img/`
- `assets/videos/`
- `.nojekyll`
- `.gitignore`

GitHub Pagesで公開する場合:

1. GitHubで新しいリポジトリを作成
2. このフォルダ内のファイル一式をアップロード
3. `Settings` → `Pages`
4. `Build and deployment` の `Source` を `Deploy from a branch` にする
5. `Branch` を `main`、フォルダを `/root` にして保存
6. 数分後に表示されるURLへアクセス

注意:

- `index.html` がルート直下にある状態でアップロードしてください。
- 動画は `assets/videos/hero-promo.mp4` に入っています。
- GitHub Pagesでは問い合わせフォームなどのサーバー処理は動きません。必要な場合は外部フォームやEC側URLへリンクしてください。

## 1. LP全体の設計意図

「沖縄の自然と祈りを宿すアクセサリー」という思想を、商品一覧より先に空気感として伝える1ページLPです。世界観ボードから抽出した深緑、生成り、乳白色、鈍い金、細い罫線、余白、標本のような静けさを全体設計に反映しています。

## 3. 見出し・本文・CTA

- FV: 「沖縄の自然と祈りを、身にまとう。」 CTAは「商品を見る」「ブランドコンセプトを見る」
- 商品ギャラリー: 「自然の気配を、日々のそばへ」 各カテゴリに「商品を見る」
- ブランドコンセプト: 「沖縄の自然と祈りを宿すものづくり」
- 購入導線: 「身につけたい祈りを、選ぶ」
- オーダーメイド: 「想いをかたちにする、オーダーメイド」 CTAは「オーダーメイドを相談する」
- 店舗: 「お手に取れる場所」 CTAは「取扱店舗を見る」
- イベント: 「展示会・ワークショップのお知らせ」 CTAは「Instagramで最新情報を見る」

## 4. 各セクションで使う画像の種類

- ヘッダー/フッター: ロゴ画像
- FV: ネックレス/商品ディスプレイの象徴画像、貝パーツの質感画像
- 商品ギャラリー: カテゴリ別商品写真
- ブランドコンセプト: 貝の質感写真、ロゴ小配置
- 購入導線: 4カテゴリの導線画像
- オーダーメイド: ペンダント/パーツの寄り写真
- 店舗/イベント: 商品ディスプレイ写真

## 5. ロゴ画像の配置指示

ロゴはヘッダー左上、FV内の小さなブランドサイン、コンセプト付近の余白、フッターに配置。縦横比を保持し、白背景は生成り背景と馴染ませるために余白と `mix-blend-mode` を使っています。

## 6. 世界観ボードの参照指示

世界観ボードはファーストビューには掲載せず、配色、背景質感、細い罫線、余白、明朝系の見出し、植物標本的な静けさの基準として参照しています。

## 7. 商品写真の分類・配置指示

- `necklace-display.png`: ネックレス、FV象徴画像、イベント補助
- `ring-display.png`: 指輪、店舗/展示補助
- `jifa-hand.png`: ジーファー / かんざし
- `aroma-texture.png`: アロマの仮置き素材画像
- `shell-pieces.png`: ブランドコンセプト、質感イメージ
- `pendant-pieces.png`: オーダーメイド、素材/パーツイメージ

## 8. PC / スマホのレイアウト方針

PCは余白を広く取り、FVは左コピー右画像、商品は2列、購入導線は4列で構成。スマホは画像と本文の休符を保つため、FVは画像上、コピー下、各カードは1列で大きく見せます。
